Tools & Stack

The Waylight way is to favour simple, reliable tools that minimise complexity and long-term maintenance.

Wherever possible, the aim is to prioritise open standards, lightweight systems, and widely supported platforms that organisations can manage realistically and economically.

All of these approaches are achievable in-house, and this website explains the methods openly. Waylight provides practical implementation support for organisations that want results without adding internal workload. Demonstration builds are available on the Workbench.

Tools change. Principles remain.

Build Principles

What Waylight websites prioritise

Static websites, minimal dependencies, fast loading speeds, accessible design, and long-term maintainability.

Component Choice
Structure HTML
Styling CSS
Interactivity Minimal JavaScript
Hosting Static hosting platforms

This translates to dependable websites that avoid weekly break-fix cycles.

Order Over Replacement

Bring order to mixed environments

Most organisations already use a mixture of platforms. Waylight support often focuses on simplifying and improving usage rather than replacing everything.

Common environments include Outlook, Teams, Google Docs, Dropbox, Excel spreadsheets, and legacy websites.

Avoiding Overload

Systems generally avoided

As a general rule, Waylight avoids systems that introduce unnecessary complexity or long-term maintenance burden for small organisations.

Examples include heavily plugin-dependent websites, proprietary systems with expensive subscriptions, and over-engineered tech stacks.

Exploration

Areas under active review

Current exploration includes AI-assisted workflows, practical automation for small organisations, and improved accessibility tools.

Waylight Stack Diagram

Build -> Repo -> Deploy -> Host

1. Build
HTML, CSS, and minimal JavaScript are developed in Visual Studio Code.
2. Repo
Code is versioned in GitHub for traceable change history and rollback control.
3. Deploy
Automated deployment runs from GitHub to a static platform with minimal moving parts.
4. Host
Netlify or equivalent static hosting serves the live website with low maintenance overhead.
